# 1. 용어집 (Glossary)

serv
: 지속적으로 실행되며 서비스를 제공하는 서버 프로그램

host
: 하나 이상의 서버(serv)가 실행 중인 머신

command
: 서버 또는 클라이언트에 대한 명령 요청
(atcommand, script command, packet request)

interface
: 명령 목록을 제공하는 클래스 또는 모듈

# 2. 소개 및 에뮬레이션 구조

rAthena는 Ragnarok Online의 공식 서버인 AEGIS를 에뮬레이션한 프로젝트이다.

AEGIS는 다음 4개의 서버로 구성된다.

account
: 계정 및 로그인 처리

char
: 캐릭터 데이터 저장

inter
: 서버 간 통신

map
: 실제 게임 처리

rAthena에서는 inter 서버가 char 서버에 통합되어 있다.

구조:

login-server
├─ char-server
│   ├─ map-server
│   ├─ map-server
│   └─ ...

login-server는 최대 30개의 char-server를 관리할 수 있으며,
각 char-server는 최대 30개의 map-server를 관리할 수 있다.

login-server와 map-server는 직접 통신하지 않으며,
반드시 char-server를 거쳐 통신한다.

---

# 3. 인터페이스 및 통신

rAthena에는 3가지 통신 방식이 존재한다.

## 1) 서버 ↔ 서버

예:

login ↔ char
char ↔ map

이를 inter-server 통신이라 한다.

## 2) 서버 ↔ 클라이언트

게임 클라이언트와 주고받는 패킷 통신.

예:

* 이동
* 공격
* 스킬 사용
* 채팅
* 아이템 사용

## 3) 서버 ↔ 콘솔

서버 콘솔에서 사용하는 명령.

예:

* @reloadscript
* @serverinfo

현재 패킷을 사용하지 않는다.

또한 스크립트(script)와 atcommand 역시 서버와 사용자를 연결하는 인터페이스로 취급된다.

# 9. AtCommand 추가

함수 작성

ACMD_FUNC(test)
{
}

등록

ACMD_DEF(test)

또는

ACMD_DEF2("alias",test)

제한 옵션

1 = 콘솔 사용 제한
2 = 스크립트 사용 제한

---

# 10. Script Command 추가

함수 작성

BUILDIN_FUNC(test)
{
}

등록

BUILDIN_DEF(test,"i")

또는

BUILDIN_DEF2(test,"alias","i")

인자 타입

i = int
s = string
v = variable
l = label
r = reference
? = 선택 인자 1개

* = 가변 인자
  null = 인자 없음

자주 사용하는 함수

script_hasdata()
script_getdata()
script_getnum()
script_getstr()
script_getref()
script_getfuncname()
script_pushint()
script_pushstr()
script_isstring()
script_isint()

이 함수들은 BUILDIN_FUNC 구현 시 스크립트 스택의 데이터를 읽고 쓰는 데 사용된다.
